# YakLab / Yakit 学习与靶场生态概览

## 这是什么
YakLab 是 Yaklang 生态中面向学习与实战演练的一组资源集合，目标是让使用者通过“教程 + 靶场 + 插件/工具实操”的方式提升 Web 安全技能，并在真实风格场景中完成漏洞复现、插件测试与方法论沉淀。

YakLab 的内容大体可分为三块：
- 教程：Yakit 使用教程、Yak 语言教程、常见漏洞讲解
- 靶场：基础漏洞（OWASP Top 10）、逻辑漏洞（业务逻辑场景）、组件漏洞（常见组件/中间件漏洞）
- 社区协作与持续更新：鼓励通过社区共建靶场与学习资源，并持续迭代内容

## Vulinbox 靶场（与 Yakit 联动）
Vulinbox 是 Yaklang 配套靶场体系之一，特点是：
- 覆盖多种漏洞类型（如 SQL 注入、XSS、命令注入等），用于学习与插件测试
- 通过 Yakit 的实验性功能可快速部署、启动与管理
- 靶场应通过管理器正确关闭，避免后台进程残留导致资源浪费

## 从“学习靶场”到“工程化能力”的迁移
把 YakLab 的“学习路径”迁移到 AutoPentestAI 的工程化能力，建议按以下闭环做沉淀：
1. 复现：在靶场中完成漏洞的可复现验证（优先非破坏性）
2. 取证：沉淀证据（请求/响应片段、日志、截图路径）
3. 固化：把“验证步骤/排障要点/修复建议”写入 Playbook（`data/playbooks/`）
4. 知识化：把“原理/通用检测点/安全边界”写入 Knowledge（`data/knowledge/`）
5. 自动化：把“可重复的探测/枚举/模糊测试流程”固化为 Tool（`config/tools/*.yaml`）或 Skills（`data/skills/`）

## 安全边界（重要）
YakLab 的材料中包含大量“复现/测试”内容。将其引入生产型平台时务必保持边界：
- 仅在授权与隔离环境中使用（靶场/测试环境）
- 复用内容优先落在“验证/排障/取证/修复”而不是“可直接攻击的武器化脚本/载荷”
- 需要联网抓取资料时，严格禁止访问 localhost/内网地址，避免被诱导做 SSRF 式内网探测
