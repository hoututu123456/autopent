# YakLab Vulinbox 靶场实践（用于漏洞复现与插件测试）

## 适用场景
- 用于学习常见 Web 漏洞与复现流程
- 用于测试安全插件/检测规则的有效性
- 用于把“复现步骤/证据/修复建议”沉淀到 Playbook 与 Knowledge

## 安装与启动（Yakit）
1. 打开 Yakit
2. 进入：设置 → 实验性功能 →（靶场）Vulinbox
3. 点击安装靶场，等待安装完成
4. 安装完成后，关闭靶场管理器标签页，再重新进入 Vulinbox 管理页
5. 启动靶场并根据提示等待出现 “VULINBOX RUNNING IN …” 类似提示

注意：
- 靶场关闭请在 Vulinbox 管理器页面内关闭，不要直接关闭浏览器标签页，避免后台进程残留
- 如遇安装/启动问题，优先检查网络、权限与 Docker/运行时依赖（视 Yakit 实现而定）

## 复现流程模板（建议在 AutoPentestAI Playbook 中使用）
对每个靶场点建议按统一模板沉淀：
1. 资产与入口
   - 入口 URL / 参数 / 功能点
   - 影响范围（是否鉴权、是否跨用户）
2. 非破坏性验证步骤
   - 最小化请求/操作步骤
   - 预期响应特征（状态码、关键字段、行为变化）
3. 证据采集
   - 请求/响应片段（必要字段）
   - 日志/截图路径（如有）
4. 原理说明（高层）
   - 漏洞成因与触发条件
   - 常见误报/漏报点
5. 修复建议
   - 代码修复建议
   - 配置/部署层面加固建议
6. 回归验证
   - 修复后复测步骤与判定标准

## 插件测试方法（面向“检测插件/规则”）
- 明确插件目标：它要检测的“事实”是什么（版本、行为、响应特征、配置缺陷）
- 设计对照组：同功能无漏洞样本/修复后样本，用于判断误报
- 输出证据闭环：插件输出必须可复核（请求、响应、截图/日志定位）
- 把“检测规则的边界条件”写入 Knowledge（避免下次重复踩坑）
