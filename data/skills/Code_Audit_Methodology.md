# 代码审计方法论（快速落地版）

## 目标
在有限时间内最大化发现高风险问题：鉴权缺陷、输入校验缺陷、注入链路、敏感信息泄露、权限边界破坏与不安全的加密用法，并输出可复现证据与修复建议。

## 建议流程
1. 快速规则扫描：先用规则扫描定位“明显风险点”（密钥泄露、危险 API、禁用 TLS 校验等），再人工复核上下文。
2. 入口与权限建模：枚举所有外部输入入口（HTTP/RPC/CLI/队列/定时任务），梳理身份与权限模型（用户、角色、租户、资源）。
3. 数据流追踪：对“输入 → 处理 → 输出/存储”做链路追踪，重点关注拼接 SQL/命令/模板/路径/序列化等场景。
4. 关键控制点核查：
   - 认证：登录、会话、token 生成与校验、MFA、重置口令
   - 授权：对象级/功能级鉴权、租户隔离
   - 输入校验：白名单、长度限制、编码/解码边界
   - 加密：随机数、密钥管理、TLS 校验、敏感字段存储
   - 日志：是否记录敏感信息、是否可被注入污染
5. 证据与修复：每个问题必须给出精确定位（文件/行号/函数路径）与可落地修复建议（包含代码级或配置级动作）。

## 常见高危点速查
- 命令执行：`os.system`、`subprocess.*(shell=True)`、拼接命令字符串
- 反序列化：不可信输入反序列化（pickle/yaml.load 等）
- SQL 注入：拼接 SQL、ORM 的 raw 查询、动态表名/列名
- 路径穿越：拼接文件路径未做白名单与规范化
- SSRF：可控 URL 的服务端请求，未做内网/协议限制
- XSS：前端 `innerHTML`、后端模板渲染未转义
- 密钥泄露：配置文件、日志、前端打包产物中出现 key/token

## 产出模板（建议）
每条漏洞输出：
- 标题、风险等级、影响范围
- 复现步骤（非破坏性优先）
- 证据（请求/响应片段、日志片段、代码定位）
- 原理（高层解释）
- 修复建议（可执行、可验证）
